使用教程
========

系统模型
--------

易科引擎，在下文中简称为引擎，俗称八卦炉，核心模型，可以装载域空间，模拟域空间的
演化，设置域空间的使命和目标，控制域空间的演化，可以暂停/单步/动画/连续等多种模
式显示演化过程，并提供多视图显示域空间的结构和关系。

思考者，操纵引擎，通过易擎的交互界面进行控制，完成下列任何任务的组合

* 观察域空间的演化状态
* 设置需要达成目标，并记录目标实现的过程
* 设置不同的域空间初始条件，自动组合所有的可能的域空间演化结果

思考者首先装入一个域空间，设置域空间的相关属性，设置引擎的相关属性，然后域空间开
始演化。

每一个域空间的演化，就是按照不同层次，进行的各种动作。每一个动作都必须有某一个域
空间发出，称之为主动者，然后改变一个或者多个域空间，称之为从动者。

每一个改变（爻）的产生可能来源于

* 操作者: 操作人员使用某一个域空间的功能，这个功能会对相关域空间进行改变
* 主动者: 当前域空间的状态满足一定的条件，从而发起了相关事件，而产生的行为
* 映射者: 通过映射空间，在每一个时间粒度内把主动空间的变化映射成为从动空间的变化

当一个域空间被操作者暂停演变之后，其所有的子空间也自动停止演变。

引擎主要的功能包括

1. 介绍某一个域空间，指令 introduce ，用于了解概念。

2. 学习和测试一个域空间，指令 learn / test，用于展示域空间的全生命周期，测试域空间的行为和功能

3. 观察一个域空间，指令 watch / unwatch，可以同时观察多个

4. 关注一个域空间，指令 focus ，在同一个层次只能有一个空间是被关注的，不同层有映
   射空间关联的其他空间同时也会被关注

5. 深入一个域空间，指令 zoomin / zoomout，进入域空间更具体的结构，可以同时显示缩略图和详细图

6. 单步/连续/暂停/后台 指令 step / continue / pause / go 某一个空间的演化

7. 设置事件和目标，指令 event / target 达成之后暂停演化，target 等价于事件行为是暂停

8. 记录域空间的演化过程，使用动画或者日志的方式进行记录，指令 record / log

9. 和域空间进行交互，指令 talk ，唤起域空间的相关功能

10. 装载域空间，指令 load / unload

11. 设置域空间，指令 set

12. 睡眠和唤醒域空间，指令 sleep / wakeup

13. 显示不同空间的映射关系，指令 map / unmap

14. 使用不同的视角观察域空间，指令 view

域空间分为三个不同的层：

1. 物理层，也就是现实世界，使用实体构成的所有域空间。layer = 0
2. 文明层，是以人类思想为生存空间而构成的域空间，包括各种原理图等。layer = 1，也
   可以有更高层 layer > 1
3. 混沌层，构成现实世界的不可测实体，layer = -1，也可以有更高层 layer < -1。混沌
   层是物理层是通过无限关联进行映射的，所以两者是不可相互度量和比较的。

每一个层中，又可以分为两个层，是通过放大和缩小进行关联的，是一种通过有限的关联，
两者可以相互可以比较的：

1. 宏观图
2. 微观图

每一个域空间可以和另外域空间进行交谈（指令 talk），接收方听到之后，会根据交谈的
内容（Action），进行相应的行为。首先根据内容对应的行为名称，和相应的参数，查找自
己是否具备执行的能力，这里域空间会首先查找自己是否具备这样的能力（doActionXXX），
同时参数类型要匹配。如果域空间自身没有这样的能力，会查找所有子域空间，如果找到，
那么执行。如果找不到，要告诉对方，无法执行。

引擎有三种工作模式：

1. 空闲
2. 演化，用于动态显示域空间的变化
3. 学习，用于组合域空间的各种可能性

引擎可以设置各种不同的使命，使命就是判断域空间是否满足某一种条件，符合这个条件我
们就称之为使命完成。

使命完成之后默认行为是演化暂停，可以观察记录域空间的行为。

有一个特殊的使命是总是返回真，可用于单步跟踪域空间的演化。

爻类型
~~~~~~

引擎内部使用爻来描述每一个动画步骤，爻可以分为动画爻和控制爻

动画爻用于描述唤起空间的行为

* 发起者，空间或者实体，可以直接是Vue对象实例，也可以是条件组合，例如 计算机的硬
  盘，文件 foo.exe 等
* 动作行为名称，通过媒介 Action 对象进行调用
* 参与者（参数）
* 演示选项开关，可以覆盖全局选项

控制爻是控制引擎的显示，它主要用于

* 创建不同的显示布局
* 切换不同的显示布局
* 关闭显示布局
* 打开和关闭某一个动画演示选项
* 设置观察焦点
* 控制显示操着者视图
* 控制显示缩略图
* 选中显示视图/取消选中视图/放大缩小视图
* 显示空间的关联视图
* 创建空间的显示视图/关闭空间的显示视图


使用方法
--------

装载引擎之后，显示开始页面，用于设置引擎的选项，点击开始之后引擎运行。

引擎运行之后默认是单窗口模式显示主域空间，有主域空间提供的功能菜单。例如，计算机
域空间提供的功能包括

* 装载可执行文件
* 查看进程执行
* 查看组件的结构（空间），使用不同视角，对比不同分辨率的视图
* 查看组件的功能（时间），例如磁盘读文件等，使用不同视角，多个视图同时显示
* 查看不同选项对整体的影响，修改选项，显示和该选项所有相关的结构和功能变化

全局绑定的行为，只要引擎运行，这些行为就一直存在

* 切换到全屏模式，快捷键 F12 全局工具栏按钮，ESC 退出全屏
* 切换到视窗管理，快捷键 Shift + TAB 全局工具栏按钮，ESC 退出视窗管理

引擎有三种工作模式：

* 初始化模式，Idle，初始化
* 演化模式，Living，观察演化过程
* 学习模式，Learning，了解域空间，学习域空间的功能

域空间的状态分为两类:

* 运行，正在进行演化
* 其他非运行状态，例如，就绪，暂停，结束，出错等

在运行状态，一般显示全局工具栏，可以单步/连续等控制演化过程

* 在连续运行状态，ESC 用于退出动画，返回上一级操作，空格键用于暂停
* 在就绪和暂停状态，空间键用于单步执行，会进入任何下一级的明细，回车键只在相同层
  单步运行，Shift + 空格 进入连续动画模式，shift + Enter 进入后台运行模式
* 在结束和出错之后，空格 和 回车键可以重新进入就绪

在学习模式，可以使用标准域空间视图，装载任意域空间，设置域空间的属性，装入新的域
空间，多角度观察域空间，学习和了解域空间的生命周期

初始化设置模式
~~~~~~~~~~~~~~

初始化域空间，例如创建并装入一个磁盘文件，设置计算机的自动启动程序等

在空闲设置模式，左边是主域空间的树形结构图，显示所有子空间的名称

* 点击空间名称后面的开关可以打开选中空间的详细视图，如果有的话
* 点击修改按钮可以修改空间的属性
* 移动到省略号上可以显示空间对应的所有可用视图，点击可以打开对应的视图

工具栏的按钮

* 创建按钮 为当前空间创建子组件
* 动画演示 可以进入动画演示模式
* 学习研究 可以进入学习研究模式
* 目标设定 设置目标
* 选项设置 引擎选项设置

* 缩略图模式
* 关联视图模式

对于显示在右侧区域的实体对象，可以进行如下操作

* 点击域空间的观看图标，或者回车键或者双击进入下一级详细的视图，相当于放大
* ESC 返回上一级视图，或者 双指左右滑动，可以在上下级视图进行切换
* 单击可以选中/取消选中，Shift + 单击可以在多选状态下切换选中状态
* 点击工具栏按钮多视图模式按钮，或者 Ctrl + 单击选中视图可以进入关联视图模式，再次重复则切换不同的视角
  * 只显示域空间的主视图
  * 三维排列所有可用视图
* 在关联视图模式
  * Shift + 单击，使用选中视图进入主视图模式
  * 鼠标左右滑动也可以切换不同视图
  * 单击，可以选中指定视图
  * ESC 退出关联视图模式，并使用当前选中的视图进入单视图模式，但是不会设置选中视图为主视图
  * 回车 则进入单视图模式，同时设置选中视图为主视图
* 工具栏按钮缩略图模式，用于关闭和显示上级缩略图，如果有多级缩略图，则依次显示，然后依次关闭

动画演示模式
~~~~~~~~~~~~

在动画演示模式，使用的是动画控制语言来显示空间的演变过程，是爻的体现。

每一个空间的行为都可以最终的表达为

* 一个发起者，开始执行某一个功能
* 多个参与者，参与执行这个功能
* 每一个功能，总是可以通过下面的某一种方式表达出来
  * 空间（实体）被创建出来，能够使用动画的方式把创建表达出来，同时会显示出谁是创
    造者，这是其他的动画方式一般不会有的特点
  * 实体被毁灭，把毁灭的原因也能够显示出来，例如破坏者，或者自身原因等
  * 实体进行了移动，从某一个位置到另外一个位置
  * 实体的属性发生了变化，属性的变化可以通过文字/颜色/形状等的改变来体现，也可以
    通过一种动画方式来表示

思考者可以通过下列开关控制动画显示的方式

* 是否显示发起者/参与者，如果是则在动画的过程中显示出发起者
* 是否显示功能的动画过程，如果是则动画显示，否则直接实现功能
* 是否显示当前实体的关联视图，例如文明层的视图，可以多视角查看实体的存在形式
* 是否显示缩略图，当有些空间的组成部分太小，无法和空间本身同时显示的时候，会把空
  间缩小，浮动显示在旁边，而把实体的组成部分放大，在缩略图上使用标示点显示组成部
  分的变化和位置。
  缩略图可以是多级，当进入一个子组件时候，显示父组件的缩略图；当再次进入下一级别
  的组件时候，缩略图就有两级显示级别
* 是否进入细节展示，如果当前功能有很多子功能组成，那么是否进入这些子功能进行动画
  显示。如果是进入子功能，展示子功能的实现过程，如果否，则不展示执行过程，而是直
  接显示执行结果
* 放大或者缩小当前视图，当放大或者缩小到一定程度，会使用另外一种视角显示
* 移动或者旋转视图
* 局部放大视图，显示某一个细节，这种方式不会改变显示视图关系，不会显示缩略图
* 选中一个或者多个视图
* 显示思考者观察的焦点，也就是当前眼睛注视的视图区域或者视图对象

学习研究模式
~~~~~~~~~~~~

可以学习某一个空间的功能，这时候会把功能使用不同参数的动画行为，依次的展示出来。

可以学习一个空间的选项，这时候会把和选项相关的所有功能都依次的使用动画的方式展示
出来

操作方式
* 选中空间
* 点击学习研究按钮
* 弹出空间的所有选项和功能列表
* 选中需要学习的功能或者选项
  * 可以使用全覆盖的方式自动生成全部功能组合
  * 可以自己设置功能的输入
  * 可以查看空间的全生命周期
* 点击下一步，显示所有的功能组合，点击下一步
* 进入动画显示状态，依次显示各个功能和选项的作用
